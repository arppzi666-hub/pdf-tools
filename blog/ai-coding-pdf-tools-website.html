How AI Helped Me Build a PDF Tool Website in One Day

June 7, 2026 · 8 min read

I built pdftools.fun — a complete online PDF toolkit with 8 utilities, Chinese/English language switching, and Google AdSense integration — in a single day using AI coding assistants. Here's the exact process, prompts, and lessons learned.

The Stack

  • AI Assistant: Claude Code with DeepSeek V4 Pro
  • Frontend: Single-file HTML/CSS/JS (no framework)
  • PDF Libraries: pdf-lib, PDF.js, JSZip, FileSaver.js, mammoth.js, docx
  • Hosting: GitHub Pages (free, global CDN)

Step 1: Define the Scope

Instead of building one tool at a time, I listed all 8 tools upfront:

  1. Merge PDFs
  2. Split PDF
  3. Compress PDF
  4. PDF to JPG
  5. Rotate PDF
  6. Unlock PDF
  7. Word to PDF
  8. PDF to Word

This scoping prevented scope creep and gave the AI a clear target.

Step 2: Generate the Base

The prompt was simple but specific:

Build a single-file HTML website with 8 PDF tools.
All processing must be client-side. Use CDN libraries.
Add Chinese/English i18n. Include Google AdSense slots.
Mobile responsive. Clean modern UI.

The AI generated ~500 lines of working code in one shot.

Step 3: Debug Iteratively

The hardest bugs were NOT code logic — they were environmental:

  • CDN script blocking: Synchronous script tags blocked all inline JS on slow mobile connections. Fixed by moving UI code BEFORE CDN scripts.
  • Mobile unresponsive: Event listeners weren't attaching because deferred scripts executed after inline ones. Fixed with DOMContentLoaded wrapper.
  • PDF to Word formatting loss: Text extraction via pdfjsLib discarded all formatting. Fixed by rendering pages as images and embedding them as ImageRun in docx.

Step 4: i18n Architecture

Rather than using a heavy i18n library, I used a simple data-driven approach:

var _i18n = {
  en: { merge: { title: 'Merge PDF Files', ... } },
  zh: { merge: { title: '合并 PDF', ... } }
};
function _t(key) { ... }
function _applyLang() { ... }

All translatable elements use data-i18n attributes. The language preference persists via localStorage.

Key Takeaways

  1. Define everything before coding. A clear list of 8 tools prevented feature creep.
  2. Test on real mobile devices. Desktop Chrome can't catch CDN blocking issues.
  3. Use AI for debugging, not just generation. The AI read error logs, identified root causes, and proposed fixes faster than manual debugging.
  4. Single-file architecture wins for simple projects. No build step, no deployment complexity.

What's Next

I'm adding more tools (PDF to Excel, OCR for scanned PDFs) and building a blog (you're reading it) for SEO. The entire site will remain free and ad-supported.
